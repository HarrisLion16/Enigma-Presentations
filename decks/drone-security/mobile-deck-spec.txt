GLOBAL SPECIFICATIONS
=========================================================

TARGET VIEWPORTS:
- iPhone landscape: ~844x390
- Android landscape: ~915x412
- Usable content area after padding: ~812x342 (iPhone) / ~883x364 (Android)
- Design to the smaller viewport (844x390) as the constraint

PORTRAIT BEHAVIOR:
- Show full-screen "Rotate your device" overlay
- Same treatment as the existing desktop deck
- Trigger: orientation === 'portrait' on screens <= 1024px wide

SLIDE CONTAINER:
- Each slide: width 100vw, height 100vh (use 100dvh for iOS safe area)
- overflow: hidden on each slide (no scroll within a slide)
- background: #0A0E17 per slide

SWIPE TRACK:
- Outer wrapper: overflow hidden, width 100vw, height 100dvh
- Inner track: display flex, flex-direction row
- Each slide is flex: 0 0 100%, width 100%
- Position slides via transform: translateX(-N * 100%) on the track
- Transition: transform 300ms ease-out
- Touch swipe detection:
  - Track touchstart X, touchmove X, touchend X
  - Threshold: 50px minimum swipe distance
  - Swipe left (negative deltaX) = next slide
  - Swipe right (positive deltaX) = previous slide
  - During swipe, move track by finger delta (rubber-band feel)
  - On release, snap to nearest slide

EDGE PEEK (optional, subtle):
- No edge peek. The dot indicators are sufficient for discoverability.
- If implemented later, show 8px of next slide's left edge with a dim gradient overlay.

SLIDE INDICATOR DOTS:
- Position: fixed, bottom 8px, left 50%, transform translateX(-50%)
- Container: display flex, gap 4px, z-index 100
- Each dot: width 6px, height 6px, border-radius 50%
- Active dot: background #FFD930 (gold)
- Inactive dots: background rgba(255,255,255,0.15)
- Transition: background 300ms ease
- No tap interaction on dots (too small to be useful)

LOGO:
- Enigma logo PNG, position absolute, top 8px, left 12px
- Height: 16px, width auto
- Opacity: 0.7
- z-index: 10

=========================================================
IMPLEMENTATION NOTES FOR FRONT-END DEVELOPER
=========================================================

1. FILE STRUCTURE: Single standalone HTML file. All CSS inline in <style>.
   All JS inline in <script>. No external dependencies except Google Fonts (Inter).
   The Enigma logo can be embedded as base64 (same as desktop deck) or referenced
   as enigma-logo.png from the same directory.

2. SWIPE ENGINE: Use vanilla JS touch events. No library needed.
   - touchstart: record startX, startTime
   - touchmove: calculate deltaX, apply transform to track (follow finger)
   - touchend: if |deltaX| > 50px and elapsed < 300ms, navigate. Otherwise snap back.
   - Prevent vertical scroll during horizontal swipe.

3. VIEWPORT META: <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
   This prevents zoom (we want fixed layout).

4. CSS CUSTOM PROPERTIES: Reuse the same variable names as the desktop deck for
   consistency. Override values as needed.

5. TESTING PRIORITY: iPhone Safari (most restrictive), Chrome Android, Firefox Android.
   Test with DevTools mobile emulation first, real devices second.

6. PERFORMANCE: No base64 logos repeated 14 times. Define once, reference via CSS or
   a single DOM element positioned fixed. This cuts file size dramatically.

7. The file should be named: drone-security-mobile.html

END OF SPECIFICATION
